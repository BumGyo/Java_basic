* 소켓 네트워크 프로그래밍 절차
-> 서버 역할을 하는 프로그램
-> 클라이언트 역할을 하는 프로그램
   -> 클라이언트/서버 모델을 구현

-> 서버 역할을 하는 프로그램을 작성 중
1) 서버 소켓 객체를 생성해서 클라이언트에서
   서버 프로그램을 찾을 수 있도록 해줍니다!!
   -> 포트 번호를 생성

2) 포트 번호를 클라이언트에게 알려주어야 함
   -> 컴퓨터에 특정 디렉토리 안에
   새로운 파일을 만듭니다!!
   -> 속성 파일(프로퍼티스: Properties)을 사용
      : 기본 구조가 미리 정해져 있는 파일
      : 키(Key)와 값(Value)의 쌍으로 표현


* P2P 모델을 구현 : Point To Point
  -> 서버와 클라이언트 역할이 정해져 있지
  않는 모델
